Erklaerung zu Export:- Einstellen des gewuenschten Export-Formates ueber das Popup-Menue- Es wird immer der Inhalt des obersten Windows exportiert.- Als Default werden die in "Darstellung" ausgewaehlten Punkte uebertragen.- Man kann jedoch links Items selektieren und mit Kopieren ("==>" oder Taste  "K") diese nach rechts kopieren. War rechts etwas selektiert, so wird es  ersetzt.- Selektiert man etwas rechts und drückt "L", so werden die selektierten  Punkte entfernt.Erklaerungen zum Import:- gleiche Formate wie beim Import- Momentan werden _alle_ Textdateien in der Standard Fileauswahlbox dargestellt.- Das Format wird automatisch erkannt.- Nach der Standard Fileauswahlbox wird die Datei komplett eingelesen.- Die Datensaetze werden im linken Bereich dargestellt.- Mit "<" bzw. ">" kann man durch die einzelnen Datensaetze blaettern. Es  funktionieren auch die Cursortasten.- Wird auf eine entsprechende Zeile des Datensatzes geklickt, so wird der Pfeil  entwedern grau oder schwarz. Ist er grau, so wird diese Zeile nicht importiert.  Bei den schwarzen Pfeilen muss rechts noch eine Zuordnung erfolgen, d.h. an  welche Position soll die Zeile kopiert werden. Dies geschieht durch das  Anklicken z.B. der ersten Zeile gefolgt von einem Klick (mit gedrueckter Option-  Taste) auf die zeite Zeile. Voilà, die Zeilen wurden vertauscht.- Import impoertiert schliesslich die Datensaetze.- Datensaetze werden immer ins oberste Fenster importiert.Die Suchsprache wird im Programm an allen Ecken und Enden intern verwendet. Sieermoeglicht das Finden von bestimmten Datensaetzen. Welche Datensaetze gesuchtwerden, das wird durch die Suchsprache beschrieben. Gleichzeitig kann man durchdie Suchsprache alle gefundenen Datensaetze zaehlen, selektieren, finden,markieren, ein Unterprogramm aufrufen, u.s.w. Der Anwender hat mit der Suchsprache_nichts_ direkt zu tun.Momentan ist sie so definiert:Als ersten Befehl kann ein "c12=" stehen, wobei das "12" fuer eine Zahl von00 bis 09 steht. Diese interne Funktion wird aufgerufen, wenn der gesamteAusdruck true ist.CLFCount		00 : CLCount-Zaehler hochzaehlen					 (NUR intern verwendbar)CLFFound		01 : Found-Flag setzen/loeschenCLFListAdd		02 : Element an CLFList anhaengen					 (NUR intern verwendbar)CLFCallSub		03 : Funktion für ListH aufrufen (IMMER wieder auf "Return" setzen!!!)					 (NUR intern verwendbar)CLFSelect		04 : Selektiert-Flag setzen/loeschenCLFMarked		05 : Markiert-Flag setzen/lieschenCLFDisplaymode	06 : Displaymode-Flag setzen/loeschenCLFList1		07 : Typ auf Telefondata 1 setzenCLFList2		08 : Typ auf Telefondata 2 setzenCLFList3		09 : Typ auf Telefondata 3 setzenEs folgt nun der Ausdruck:Rechenprioritaet:oben	Variablen, Klammern, Funktionen		"n" fuer NOT		"a" fuer ANDunten	"o" fuer OR und "x" fuer XORDer Ausdruck darf beliebig komplex sein.Als Variablen/Funktionen gibt es folgendes:- 0		stets 0- 1		stets 1- s+|Suchbegriff|		true, wenn der Text "Suchbegriff" in dem Datensatz enthalten ist.		Das "+" gibt an, daß Gross-/Kleinschreibung nicht unterschieden wird.		Es kann entsprechend weggelassen werden.- c12	true, wenn die Funktion 12 (Nummern siehe oben) true zurueckgibt.- S		true, wenn der Datensatz selektiert ist.- F		true, wenn der Datensatz gefunden wurde.- M		true, wenn der Datensatz markiert ist.- A		true, wenn der Datensatz mit einem Alarm behaftet ist.- D		true, wenn der Datensatz 4zeilig ist.- O		true, wenn der Datensatz im alten Link-Format vorliegt.- C		true, wenn der Datensatz geaendert wurde.- X		true, wenn der Datensatz geloescht wurde.- v		true, wenn der Datensatz gemaess der Usereinstellungen (im Float) sichtbar ist.Das Anwenderlexikon:Button:Setzen		- nix selektiert 		: Begriff wird angehaengt			- Begriff selektiert 	: Begriff ersetzt die SelektierungLoeschen	- nix selektiert 		: - nix -			- Begriff selektiert 	: Selektierung wird geloeschtEinfuegen	- nix selektiert 		: Begriff wird angehaengt			- Begriff selektiert 	: Begriff wird _vor_ dem selektierten Eintrag									  eingefuegt
